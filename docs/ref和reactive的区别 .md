在 Vue3 中，`ref` 和 `reactive` 都是用于创建响应式数据的核心 API，但它们在支持的数据类型、底层实现以及使用方式上存在显著差异。以下是它们的核心区别：

1. 支持的数据类型不同
   `ref`：支持所有数据类型。它既可以包裹基本数据类型（如 `number`、`string`、`boolean`），也可以包裹引用数据类型（如 `object`、`array`）。
   `reactive`：仅支持引用数据类型。它只能用于处理对象、数组以及` Map`、`Set` 等集合类型，不能用于基本数据类型。
2. 访问与修改方式不同
   `ref`：在 JavaScript 脚本中，必须通过 `.value` 属性来访问或修改其内部值；但在 Vue 模板中使用时，Vue 会自动进行解包，无需手动添加 `.value`。
   `reactive`：返回的是原始对象的响应式代理，无论是在脚本还是模板中，都可以像操作普通对象一样直接访问和修改属性，无需 `.value`。
3. 解构赋值的响应式表现不同
   `ref`：解构后不会丢失响应式。因为解构出来的是 `Ref` 实例对象，修改其 `.value` 依然能触发视图更新。
   `reactive`：直接解构会丢失响应式。因为解构出的属性值脱离了 Proxy 代理上下文，变成了普通变量。若要保留响应式，必须配合使用 `toRefs()` 函数。
4. 整体替换对象的表现不同
   `ref`：支持整体替换。直接对 `ref` 变量重新赋值（如 `ref.value = newObj`）不会丢失响应式连接。
   `reactive`：不支持整体替换。如果直接将一个新对象赋值给 `reactive` 变量，会导致原有的响应式代理连接丢失，新对象将不再具备响应式能力。
5. 底层实现原理不同
   `ref`：对于基本类型，Vue 内部会创建一个包装器对象，通过 `Object.defineProperty` 劫持 `.value` 属性的 `get` 和 `set` 来实现响应式；对于引用类型，`ref` 内部会自动调用 `reactive` 将其转换为深层响应式对象。
   `reactive`：完全基于 ES6 的 Proxy 实现，通过创建代理对象来拦截目标对象属性的读取、修改、新增和删除等操作，实现细粒度的响应式。
   总结与最佳实践
   由于 `ref` 能够兼容所有数据类型，且避免了 `reactive` 在解构和整体替换时容易丢失响应式的隐患，Vue 官方（包括尤雨溪）强烈建议：在 `<script setup> `中，统一优先使用 `ref()`来声明所有响应式状态，除非有特定的复杂对象场景需要使用 `reactive`。

针对这类不需要响应式更新的数据，有以下几种优化方案：

1. 使用 `markRaw` 标记永不代理（推荐）
   对于像配置列表这样的大型静态数据，可以使用 `markRaw()`将其标记为原始对象。这样 Vue 就永远不会将其转为响应式，从而彻底免除 Proxy 代理带来的性能开销46。

```
# javascript 编辑
import { markRaw } from 'vue';
const firstScreen = markRaw([{ a: '11' }]);
```

2. 使用 `shallowRef` 创建浅层响应式
   如果你确实需要在某些时候修改这个数组（例如替换整个数组），但内部对象不需要响应式，可以使用 `shallowRef`。它只追踪引用本身的变化，而不递归转换内部对象，能大幅减少初始化时间。

```
# javascript 编辑
import { shallowRef } from 'vue';
const firstScreen = shallowRef([{ a: '11' }]);
```

3. 使用 `v-once` 优化渲染
   如果这个数据在模板中被渲染，且内容在首次渲染后就不再改变，可以在模板元素上添加 `v-once` 指令。这会让该子树在首次渲染后跳过后续所有的更新，减少虚拟 DOM 对比的开销。

```
# html 预览
<div v-for="item in firstScreen" :key="item.a" v-once>
  {{ item.a }}
</div>
```

4. 避免在模板中使用复杂表达式
   不要在模板中直接对 `firstScreen` 进行复杂的过滤或格式化操作，因为每次组件重新渲染时这些函数都会被调用。应使用 `computed` 属性来处理，利用其缓存特性避免重复计算。
   总结：优化的核心思路是“按需响应”。对于纯静态配置数据，坚决避免使用深层响应式（`ref`/`reactive`），改用 `markRaw` 或普通常量，可以显著降低内存和 CPU 的开销。
