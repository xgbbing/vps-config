# 缓存文件清理

.log.swp 是 Vim/Vi 编辑器因异常退出（如断电、强制杀进程）留下的‌交换文件‌，导致再次编辑时触发 E325 警告。请根据是否需恢复未保存内容选择以下方案：

核心解决步骤
‌确认无其他进程占用‌
先检查是否有其他 Vim 实例正在编辑该文件，避免数据冲突：

bash
ps aux | grep vim
若确认无活跃进程，可安全执行后续操作 。‌‌

- ‌方案一：无需恢复数据（直接删除）‌
若上次编辑内容不重要或已保存，直接删除残留的 .swp 文件即可恢复正常编辑：

bash
# 进入文件所在目录
cd /你的/文件/路径
# 删除对应的 .log.swp 文件（注意文件名前的点）
rm .log.swp
删除后重新打开原文件即可正常编辑 。‌‌

- ‌方案二：需要恢复未保存内容‌
若担心丢失上次未保存的修改，先尝试恢复：

在 Vim 报错界面输入 R 进入恢复模式，或终端执行：
bash
vim -r log
恢复成功后，立即执行 :wq 保存到新位置或原文件。
最后手动删除 .log.swp 文件，防止下次启动再次报错 。‌‌
临时快速绕过
在 Vim 报错界面（显示 O/E/R/Q/A 选项时）：

输入 ‌E‌：强制编辑（不加载 swap 内容，可能丢失未保存更改）。
输入 ‌Q‌：退出，然后按上述“方案一”手动删除文件 。‌‌
预防建议
正常退出 Vim 请使用 :wq 或 Shift+ZZ，避免直接关闭终端。
若无需备份功能，可在 ~/.vimrc 中添加 set noswapfile 禁止生成 .swp 文件（但会失去崩溃恢复能力）
